Análisis Discriminante Lineal y Cuadrático

Predicción de aprobación de estudiantes (Pass/Fail) a partir de horas de estudio y nota de examen previo, comparando la frontera de decisión recta de LDA contra la frontera curva de QDA sobre 500 observaciones.

1.1 Resumen ejecutivo

El objetivo del trabajo fue clasificar a estudiantes en Aprobado o Desaprobado usando dos variables predictoras cuantitativas: study_hours (horas dedicadas al estudio) y prev_score (nota de un examen previo). El dataset student_exam_data_new.csv contiene 500 observaciones, con una proporción de clases de 63.2% Desaprobados y 36.8% Aprobados, usada como probabilidad previa (prior) de cada modelo.

El Análisis Discriminante Lineal (LDA) asume que ambas clases comparten una misma matriz de covarianzas y por lo tanto solo puede trazar una frontera de decisión recta (un hiperplano) entre los grupos. El Análisis Discriminante Cuadrático (QDA) relaja ese supuesto, permitiendo que cada clase tenga su propia matriz de covarianzas, lo que genera una frontera curva capaz de adaptarse a relaciones de interacción entre variables.

La exploración gráfica reveló que los estudiantes aprobados se concentran en la esquina superior derecha del plano horas de estudio × nota previa — es decir, aprueban principalmente cuando ambas variables son altas simultáneamente. Este patrón de interacción es precisamente lo que un modelo estrictamente lineal no puede capturar bien, y explica por qué QDA termina superando a LDA en todas las métricas evaluadas.

Ambos modelos se implementaron en paralelo en R (funciones lda() y qda() del paquete MASS) y en Python (clases LinearDiscriminantAnalysis y QuadraticDiscriminantAnalysis de scikit-learn), obteniendo resultados coherentes entre ambos lenguajes.

Sobre los supuestos: la prueba de Shapiro-Wilk mostró falta de normalidad univariante (p < 0.001) en las cuatro combinaciones grupo × variable. Ambos modelos asumen normalidad multivariante dentro de cada clase, pero — siguiendo la literatura citada en el curso (Li, Zhu & Ogihara) — LDA y QDA son razonablemente robustos ante violaciones moderadas de este supuesto, lo cual se confirma en los resultados: ambos modelos logran una capacidad de clasificación aceptable a buena pese al incumplimiento formal.

1.3 Implementación — RStudio

Script principal en R utilizando el paquete MASS, siguiendo el procedimiento de Fisher descrito en el marco teórico del informe.

analisis_discriminante_estudiantes.R
# Carga del dataset
datos <- read.csv("student_exam_data_new.csv", header = TRUE)
names(datos) <- c("study_hours", "prev_score", "pass_fail")
datos$pass_fail <- as.factor(datos$pass_fail)
levels(datos$pass_fail) <- c("Fail", "Pass")

# Particion 70/30 con semilla fija para reproducibilidad
set.seed(123)
n <- nrow(datos)
idx_train <- sample(seq_len(n), size = 0.7 * n)
train <- datos[idx_train, ]
test  <- datos[-idx_train, ]

# Modelo LDA (asume covarianza comun entre clases)
library(MASS)
modelo_lda <- lda(pass_fail ~ study_hours + prev_score, data = train)
pred_lda   <- predict(modelo_lda, newdata = test)

# Matriz de confusion
table(test$pass_fail, pred_lda$class,
      dnn = c("Clase real", "Clase predicha"))

# Modelo QDA (covarianza propia por clase -> frontera curva)
modelo_qda <- qda(pass_fail ~ study_hours + prev_score, data = train)
pred_qda   <- predict(modelo_qda, newdata = test)
Qué hace cada bloque:
  • set.seed(123) + sample() — fija la aleatoriedad para que la partición 70/30 sea reproducible entre ejecuciones.
  • lda(pass_fail ~ study_hours + prev_score, ...) — estima los coeficientes de la función discriminante de Fisher maximizando la razón entre variabilidad entre grupos y variabilidad intra-grupos.
  • predict(modelo_lda, newdata = test) — devuelve la clase predicha ($class) y las probabilidades posteriores ($posterior) para cada observación de prueba.
  • qda() se llama con la misma fórmula, pero internamente estima una matriz de covarianzas distinta por cada clase, en vez de una matriz agrupada (pooled).

library(pROC)

roc() construye la curva ROC a partir de las probabilidades posteriores de la clase "Pass"; trainControl(method="cv", number=10) configura una validación cruzada de 10 particiones que caret::train() ejecuta automáticamente para estimar el accuracy de generalización de cada modelo (0.864 ± 0.079 para LDA, 0.936 ± 0.046 para QDA).

1.4 Implementación — Python

Réplica equivalente con scikit-learn, usada para validar de forma cruzada los resultados obtenidos en R.

analisis_discriminante_estudiantes.py
import pandas as pd
from sklearn.discriminant_analysis import (
    LinearDiscriminantAnalysis as LDA,
    QuadraticDiscriminantAnalysis as QDA,
)
from sklearn.model_selection import train_test_split
from sklearn.metrics import confusion_matrix, accuracy_score

# Carga del dataset
datos = pd.read_csv("student_exam_data_new.csv")
datos.columns = ["study_hours", "prev_score", "pass_fail"]
X = datos[["study_hours", "prev_score"]]
y = datos["pass_fail"]

# Particion 70/30 estratificada (mantiene la proporcion de clases)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=123, stratify=y
)

# Modelo LDA
modelo_lda = LDA(solver="svd", store_covariance=True)
modelo_lda.fit(X_train, y_train)
clase_lda = modelo_lda.predict(X_test)

# Modelo QDA
modelo_qda = QDA(store_covariance=True)
modelo_qda.fit(X_train, y_train)
clase_qda = modelo_qda.predict(X_test)

print("Accuracy LDA:", accuracy_score(y_test, clase_lda))
print("Accuracy QDA:", accuracy_score(y_test, clase_qda))
stratify=y asegura que la proporción 63.2%/36.8% de clases se mantenga tanto en train como en test. store_covariance=True conserva las matrices de covarianza estimadas para poder graficar después las regiones de decisión y el elipsoide de cada clase.
roc_y_cv.py
from sklearn.metrics import roc_curve, auc
from sklearn.model_selection import cross_val_score

post_lda = modelo_lda.predict_proba(X_test)
post_qda = modelo_qda.predict_proba(X_test)

fpr_lda, tpr_lda, _ = roc_curve(y_test, post_lda[:, 1])
fpr_qda, tpr_qda, _ = roc_curve(y_test, post_qda[:, 1])
auc_lda = auc(fpr_lda, tpr_lda)
auc_qda = auc(fpr_qda, tpr_qda)

cv_lda = cross_val_score(LDA(), X, y, cv=10)
cv_qda = cross_val_score(QDA(), X, y, cv=10)
predict_proba() entrega la probabilidad posterior de cada clase; roc_curve() barre todos los umbrales de decisión posibles para trazar la curva ROC completa, y cross_val_score(..., cv=10) replica en Python la misma validación cruzada de 10 particiones usada en R, confirmando la consistencia entre ambos lenguajes.

2.1 Resumen ejecutivo

A diferencia del análisis discriminante, la regresión logística no asume normalidad multivariante en los predictores: modela directamente la probabilidad del evento pass_fail = 1 mediante la función logística (sigmoide), garantizando que las predicciones caigan siempre en el intervalo [0, 1]. Los parámetros se estiman por máxima verosimilitud con glm(family = binomial(link = "logit")) en R, sobre el mismo dataset de 500 estudiantes usado en el proyecto de discriminante.

El modelo final ajustado es:

logit(P) ≈ β₀ + 1.16·study_hours + 0.13·prev_score

Ambos coeficientes resultaron estadísticamente significativos (p < 0.001), con signo positivo: más horas de estudio y mejor nota previa aumentan la probabilidad de aprobar. El diagnóstico del modelo no encontró problemas relevantes: el Factor de Inflación de Varianza (VIF) fue cercano a 1 para ambos predictores (sin multicolinealidad), la prueba de Hosmer–Lemeshow no rechazó la hipótesis de buen ajuste (p > 0.05), y el Test de Razón de Verosimilitud confirmó que el modelo completo mejora sustancialmente sobre el modelo nulo (p < 0.001).

Interpretación en odds ratio:
  • Por cada hora adicional de estudio (nota previa constante), el odds de aprobar se multiplica por ≈3.17 (IC 95%: 2.42 – 4.16) — un incremento del 217%.
  • Por cada punto adicional en la nota previa (horas constantes), el odds de aprobar se multiplica por ≈1.14 — un incremento del 14% por punto.
  • En términos relativos por unidad de medida, las horas de estudio tienen un efecto marginal más pronunciado que la nota previa sobre las probabilidades de aprobar.

2.3 Implementación en RStudio

Ajuste del modelo con glm(), diagnóstico de supuestos y evaluación predictiva completa.

regresion_logistica_multiple.R
library(car); library(caret); library(pROC); library(lmtest)
library(ResourceSelection)

datos <- read.csv("student_exam_data_new.csv", header = TRUE)
names(datos) <- c("study_hours", "prev_score", "pass_fail")
datos$pass_fail <- as.integer(datos$pass_fail)   # 0/1 para glm(binomial)

# Particion 70/30 estratificada
set.seed(123)
idx <- createDataPartition(datos$pass_fail, p = 0.70, list = FALSE)
train <- datos[idx, ]; test <- datos[-idx, ]

# Modelo nulo (solo intercepto) vs. modelo completo
modelo_nulo <- glm(pass_fail ~ 1, data = train, family = binomial(link = "logit"))
modelo_full <- glm(pass_fail ~ study_hours + prev_score,
                    data = train, family = binomial(link = "logit"))
summary(modelo_full)

# Odds ratios con IC 95%
exp(cbind(OR = coef(modelo_full), confint(modelo_full)))

# Test de Razon de Verosimilitud
lrtest(modelo_nulo, modelo_full)

# Diagnostico: multicolinealidad y bondad de ajuste
vif(modelo_full)
hoslem.test(train$pass_fail, predict(modelo_full, type = "response"), g = 10)
Qué hace cada bloque:
  • glm(..., family = binomial(link="logit")) — ajusta el modelo por máxima verosimilitud; el link="logit" es lo que aplica la transformación sigmoide.
  • exp(coef(modelo_full)) — convierte los coeficientes de log-odds a odds ratio, que es la escala interpretable como "el odds se multiplica por...".
  • lrtest(modelo_nulo, modelo_full) — compara la devianza de ambos modelos para contrastar si las covariables mejoran significativamente el ajuste frente al modelo sin predictores.
  • vif() y hoslem.test() — verifican, respectivamente, ausencia de multicolinealidad (VIF≈1) y bondad de ajuste global del modelo (p>0.05 = buen ajuste).
evaluacion_predictiva.R
prob_test <- predict(modelo_full, newdata = test, type = "response")
clase_pred <- ifelse(prob_test >= 0.5, 1, 0)

                       reference = factor(test$pass_fail, labels = c("Fail","Pass")),
                       positive = "Pass")

auc(roc_obj)

# Validacion cruzada de 10 particiones
ctrl <- trainControl(method = "cv", number = 10,
                      classProbs = TRUE, summaryFunction = twoClassSummary)
modelo_cv <- train(factor(pass_fail, labels=c("Fail","Pass")) ~ study_hours + prev_score,

                    trControl = ctrl, metric = "ROC")
type = "response" devuelve directamente la probabilidad P(Pass) en lugar del logit; el umbral >= 0.5 convierte esa probabilidad en una clase discreta para poder construir la matriz de confusión con confusionMatrix().

3.1 Resumen ejecutivo

Este es el único de los tres proyectos aplicado sobre datos reales de una entidad financiera (no un dataset académico simulado): la cartera de créditos Cash de Compartamos, con 114 clientes activos calificados internamente como B (21.9%), A (53.5%) o A+ (24.6%). El propósito es doble: explicativo — entender qué variables de comportamiento de pago separan mejor a los tres grupos —, y reclasificador — corroborar si la calificación asignada por el analista de crédito es consistente con el comportamiento real del cliente.

Se construyó un modelo LDA con 12 variables estandarizadas (montos, mora, atrasos, número de créditos vigentes, deuda en Compartamos y en la SBS, ingresos, egresos y resultado neto), tras imputar como cero los valores faltantes en variables de atraso y deuda SBS (ausencia de registro = ausencia real de esa condición). Al tratarse de 3 grupos, el modelo estima 2 funciones discriminantes (LD1 y LD2): LD1 concentra el 81.2% del poder de separación total, dominada por monto_aprobado, ingresos, n_empresas_sbs y egresos.

Supuestos verificados — y no cumplidos:
  • Normalidad univariante: las 36 combinaciones variable×grupo evaluadas con Shapiro-Wilk resultaron no normales (100%), esperable en datos financieros con colas pesadas (montos, deudas, atrasos rara vez se distribuyen simétricamente).
  • Homogeneidad de covarianzas: la prueba M de Box rechazó la hipótesis de igualdad de matrices de covarianza entre los tres grupos (χ² = 1495.48, gl = 156, p < 0.00001).
  • Decisión metodológica: siguiendo el mismo criterio aplicado al dataset Iris en el curso, se optó por mantener el LDA y reportar honestamente su desempeño real vía matriz de confusión y LOOCV, en lugar de descartarlo automáticamente por incumplir supuestos formales.
  • Multicolinealidad: se detectó correlación alta entre ingresos y egresos (r = 0.96) y entre deuda_compartamos y deuda_sbs (r = 0.75), lo que reduce la interpretabilidad individual de esos coeficientes.

datos_raw <- read_excel("data_clientes.xlsx", sheet = "rep_generacion_cash")

datos <- datos_raw %>%

# Prueba M de Box (homogeneidad de matrices de covarianza)
box_m_test <- function(data, group) {
  groups <- split(data, group)
  n_g <- sapply(groups, nrow); p <- ncol(data); n <- sum(n_g); g <- length(groups)
  S_list <- lapply(groups, function(d) cov(as.matrix(d)))
  S_pooled <- Reduce(`+`, Map(function(S, nj) S*(nj-1), S_list, n_g)) / (n-g)
  log_det_pooled <- determinant(S_pooled, logarithm = TRUE)$modulus
  log_det_each <- sapply(S_list, function(S) determinant(S, logarithm = TRUE)$modulus)
  M <- (n-g)*log_det_pooled - sum((n_g-1)*log_det_each)
  c1 <- ((sum(1/(n_g-1)) - 1/(n-g)) * (2*p^2+3*p-1)) / (6*(p+1)*(g-1))
  df <- (g-1)*p*(p+1)/2
  chi_sq <- M*(1-c1)
  list(chi_sq = chi_sq, df = df, p_value = pchisq(chi_sq, df, lower.tail = FALSE))
}
box_result <- box_m_test(datos[, vars_pred], datos$especie)

box_m_test() implementa manualmente el estadístico M de Box (chi-cuadrado aproximado) para contrastar si las tres clases comparten matriz de covarianzas — el supuesto central que diferencia a LDA de QDA. lda(..., CV = TRUE) activa la validación cruzada Leave-One-Out: cada cliente es clasificado con un modelo entrenado excluyéndolo, dando una estimación del error menos optimista que el error de entrenamiento puro.
