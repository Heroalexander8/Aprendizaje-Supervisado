Proyecto 01 · Aprendizaje Supervisado

Análisis Discriminante Lineal y Cuadrático

Aplicado a la predicción de aprobación de estudiantes a partir de horas de estudio y nota previa. Docente: Ing. Eloy Carpio Vargas.

Dataset: student_exam_data_new.csv (n=500) Split 70/30 + CV 10-fold R (MASS) & Python (scikit-learn)

01.1 — Resumen ejecutivo

Qué se hizo y por qué

El Análisis Discriminante es una técnica de clasificación supervisada que busca separar observaciones en grupos previamente conocidos a partir de variables predictoras cuantitativas. Su formulación clásica —el procedimiento de Fisher— encuentra la combinación lineal D = u₁X₁ + u₂X₂ + ... + u_kX_k que maximiza el cociente entre la variabilidad entre grupos y la variabilidad dentro de cada grupo. Bajo el enfoque bayesiano, si cada clase sigue una normal multivariante con matriz de covarianzas común (Σ), la frontera de decisión resultante es lineal: esto es el LDA. Si en cambio se permite que cada clase tenga su propia matriz de covarianzas (Σ_k), la frontera se curva: esto es el QDA.

El proyecto aplica ambos modelos a un dataset de 500 estudiantes, con dos predictores: study_hours (horas de estudio, 1–10 h) y prev_score (nota de un examen previo, 40–100 pts), para predecir si el estudiante aprueba o desaprueba (Pass/Fail). La proporción de clases es 63.2% Desaprobados y 36.8% Aprobados, valores usados como probabilidades previas (π̂) en ambos modelos.

El diagrama de dispersión reveló un patrón de interacción tipo "esquina": los estudiantes aprueban principalmente cuando ambas variables son simultáneamente altas, no cuando una compensa a la otra. Esa estructura no lineal es precisamente lo que el LDA —restringido a fronteras rectas— no puede capturar bien, y lo que el QDA sí modela mediante su frontera curva. La prueba de Shapiro-Wilk mostró falta de normalidad univariante (p<0.001) en las cuatro combinaciones grupo×variable, un supuesto teórico incumplido que, según la literatura citada en el curso (Li, Zhu & Ogihara), ambos métodos toleran razonablemente bien.

01.3 — Implementación

Scripts en R y Python

R · MASS::lda() y MASS::qda()

# 1. Carga y preparación de datos
datos <- read.csv("student_exam_data_new.csv", header = TRUE)
names(datos) <- c("study_hours", "prev_score", "pass_fail")
datos$pass_fail <- as.factor(datos$pass_fail)
levels(datos$pass_fail) <- c("Fail", "Pass")

# 2. Partición 70/30 reproducible
set.seed(123)
n <- nrow(datos)
idx_train <- sample(seq_len(n), size = 0.7 * n)
train <- datos[idx_train, ]
test  <- datos[-idx_train, ]

# 3. Modelo LDA: frontera lineal, covarianza común entre clases
library(MASS)
modelo_lda <- lda(pass_fail ~ study_hours + prev_score, data = train)
pred_lda   <- predict(modelo_lda, newdata = test)

# 4. Modelo QDA: frontera cuadrática, covarianza propia por clase
modelo_qda <- qda(pass_fail ~ study_hours + prev_score, data = train)
pred_qda   <- predict(modelo_qda, newdata = test)

# 5. Matriz de confusión y curvas ROC
table(test$pass_fail, pred_lda$class, dnn = c("Clase real","Clase predicha"))
library(pROC)
roc_lda <- roc(test$pass_fail, pred_lda$posterior[, "Pass"])
roc_qda <- roc(test$pass_fail, pred_qda$posterior[, "Pass"])
auc(roc_lda); auc(roc_qda)

# 6. Validación cruzada de 10 particiones
library(caret)
qda_cv <- train(pass_fail ~ study_hours + prev_score, data = datos,
                 method = "qda",
                 trControl = trainControl(method = "cv", number = 10))

Línea a línea: las líneas 8–12 fijan la semilla y separan 70% de las 500 observaciones para entrenar, dejando 30% (150 obs.) fuera para evaluar honestamente el modelo. lda() y qda() comparten la misma fórmula (pass_fail ~ study_hours + prev_score) pero difieren internamente en si estiman una matriz de covarianzas compartida o una por clase. predict() devuelve tanto la clase asignada ($class) como las probabilidades posteriores ($posterior), estas últimas necesarias para trazar la curva ROC. trainControl(method="cv", number=10) repite el ajuste 10 veces sobre particiones distintas para obtener una estimación de accuracy menos optimista que una única partición train/test.

Python · scikit-learn

import pandas as pd
from sklearn.discriminant_analysis import (
    LinearDiscriminantAnalysis as LDA,
    QuadraticDiscriminantAnalysis as QDA,
)
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.metrics import confusion_matrix, accuracy_score, roc_curve, auc

# Carga y partición 70/30 estratificada por clase
datos = pd.read_csv("student_exam_data_new.csv")
datos.columns = ["study_hours", "prev_score", "pass_fail"]
X = datos[["study_hours", "prev_score"]]
y = datos["pass_fail"]

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=123, stratify=y
)

# Modelo LDA (covarianza compartida)
modelo_lda = LDA(solver="svd", store_covariance=True)
modelo_lda.fit(X_train, y_train)
clase_lda = modelo_lda.predict(X_test)

# Modelo QDA (covarianza por clase → frontera cuadrática)
modelo_qda = QDA(store_covariance=True)
modelo_qda.fit(X_train, y_train)
clase_qda = modelo_qda.predict(X_test)

print("Accuracy LDA:", accuracy_score(y_test, clase_lda))   # 0.8533
print("Accuracy QDA:", accuracy_score(y_test, clase_qda))   # 0.9400

# Validación cruzada de 10 particiones sobre el dataset completo
cv_lda = cross_val_score(LDA(), X, y, cv=10)
cv_qda = cross_val_score(QDA(), X, y, cv=10)

Línea a línea: stratify=y garantiza que la proporción 63.2%/36.8% de Fail/Pass se preserve tanto en entrenamiento como en prueba. Ambas clases exponen la misma interfaz .fit() / .predict(), lo que hace directo comparar accuracy_score() entre los dos modelos con el mismo split. La diferencia de arquitectura —una covarianza compartida vs. una por clase— queda completamente encapsulada dentro de cada clase de scikit-learn.

Proyecto 02 · Aprendizaje Supervisado

Regresión Logística Múltiple

Aplicada a la predicción del resultado académico (Aprobado / Desaprobado). Docente: Ing. Eloy Carpio Vargas.

Dataset: student_exam_data_new.csv (n=500) glm(family=binomial) createDataPartition · 70/30

02.1 — Resumen ejecutivo

Del hiperplano geométrico a la probabilidad

A diferencia del LDA —que separa clases buscando una combinación lineal que maximiza la variabilidad entre grupos—, la regresión logística modela directamente la probabilidad de pertenecer a la clase positiva mediante la función logística (sigmoide), garantizando salidas en el intervalo [0,1]. El modelo se especifica como:

logit(P) = ln( P / (1−P) ) = β₀ + β₁·study_hours + β₂·prev_score

Los parámetros β se estiman por máxima verosimilitud con glm(family=binomial(link="logit")), no por mínimos cuadrados. El proyecto sigue un flujo completo de nueve etapas: exploración gráfica, análisis bivariante (pruebas t y χ²), partición 70/30, ajuste del modelo nulo vs. completo con Test de Razón de Verosimilitud, diagnóstico de multicolinealidad (VIF) y bondad de ajuste (Hosmer–Lemeshow), evaluación predictiva (matriz de confusión, ROC/AUC) y validación cruzada de 10 particiones.

Ambos coeficientes resultaron estadísticamente significativos (p<0.001), el VIF fue cercano a 1 (sin multicolinealidad relevante) y la prueba de Hosmer–Lemeshow no rechazó el buen ajuste (p>0.05): el modelo cumple razonablemente sus supuestos, a diferencia del caso de reclasificación de clientes visto más adelante.

02.3 — Implementación

Script en R

R · glm() + diagnóstico + evaluación

library(caret); library(car); library(pROC); library(ResourceSelection); library(lmtest)

# 1. Partición 70/30 estratificada
set.seed(123)
idx  <- createDataPartition(datos$pass_fail, p = 0.70, list = FALSE)
train <- datos[idx, ];  test <- datos[-idx, ]

# 2. Modelo nulo (referencia) y modelo completo
modelo_nulo <- glm(pass_fail ~ 1, data = train, family = binomial(link = "logit"))
modelo_full <- glm(pass_fail ~ study_hours + prev_score,
                    data = train, family = binomial(link = "logit"))

# 3. Significancia conjunta: Test de Razón de Verosimilitud
lrtest(modelo_nulo, modelo_full)                 # p < 0.001

# 4. Odds ratios con intervalo de confianza al 95%
exp(cbind(OR = coef(modelo_full), confint(modelo_full)))

# 5. Diagnóstico: multicolinealidad y bondad de ajuste
vif(modelo_full)                                              # ≈ 1 → sin multicolinealidad
hoslem.test(train$pass_fail, predict(modelo_full, type="response"), g = 10)  # p > 0.05

# 6. Evaluación sobre el conjunto de prueba
prob_test  <- predict(modelo_full, newdata = test, type = "response")
clase_pred <- ifelse(prob_test >= 0.5, 1, 0)
confusionMatrix(factor(clase_pred), factor(test$pass_fail), positive = "1")

roc_obj <- roc(test$pass_fail, prob_test)
auc(roc_obj)                                                  # 0.951

# 7. Validación cruzada de 10 particiones
ctrl <- trainControl(method = "cv", number = 10, classProbs = TRUE,
                      summaryFunction = twoClassSummary)
train(pass_fac ~ study_hours + prev_score, data = datos_cv,
      method = "glm", family = "binomial", trControl = ctrl, metric = "ROC")

Línea a línea: el modelo nulo (línea 8, solo intercepto) sirve de línea base para el lrtest() de la línea 12, que contrasta si añadir los dos predictores mejora significativamente la verosimilitud del ajuste. exp(coef(...)) convierte los coeficientes de la escala logit (log-odds) a odds ratios directamente interpretables. vif() del paquete car cuantifica cuánto se infla la varianza de cada coeficiente por correlación entre predictores; valores cercanos a 1 indican independencia. hoslem.test() agrupa las predicciones en g=10 deciles de probabilidad y compara los casos observados contra los esperados en cada grupo.

Proyecto 03 · Aprendizaje Supervisado

Reclasificación de Buenos Clientes

Análisis Discriminante Lineal (LDA) sobre cartera de créditos Cash – Compartamos. Docente: Ing. Eloy Carpio Vargas.

databuenosclientes.xlsx · n=114 12 variables · 3 grupos (B, A, A+) R 4.3.3 · MASS::lda()

03.1 — Resumen ejecutivo

Un caso real, con supuestos que no se cumplen

Este proyecto aplica el LDA a un problema de tres grupos (a diferencia de los dos anteriores, binarios): reclasificar a 114 clientes de la entidad financiera Compartamos según su calificación crediticia vigente (B, A, A+) a partir de 12 variables cuantitativas de comportamiento de pago y situación financiera —mora promedio, atrasos, deuda en Compartamos y en la SBS, ingresos, egresos y resultado neto, entre otras—. El objetivo es doble: explicativo (qué variables separan mejor a los clientes) y reclasificador (verificar si la calificación asignada por el analista de crédito es consistente con el comportamiento real del cliente).

Con 3 grupos, el número máximo de funciones discriminantes es mín(q−1, p) = mín(2, 12) = 2 (LD1 y LD2). Antes de estimar el modelo se estandarizaron las 12 variables (media 0, desviación 1) para poder comparar sus coeficientes en igualdad de condiciones, y se imputó como cero la ausencia de registro en tres variables de atraso/deuda, bajo el criterio de que la falta de dato refleja una condición real del cliente (no un error de captura).

A diferencia de los dos proyectos anteriores, aquí los supuestos clásicos del LDA se violan de forma contundente: el 100% de las 36 combinaciones variable×grupo resultaron no normales (Shapiro–Wilk, p<0.05), y la prueba M de Box rechazó la homogeneidad de covarianzas con un estadístico χ² = 1495.482 (df=156, p<0.00001). Siguiendo el mismo criterio que el documento base del curso aplica al dataset Iris, se optó por mantener el LDA y reportar honestamente su desempeño real en lugar de descartarlo automáticamente.

03.3 — Implementación

Script en R

R · Preparación, prueba M de Box y modelo LDA de 3 grupos

library(readxl); library(MASS); library(dplyr)

# 1. Carga y estandarización de las 12 variables predictoras
datos <- read_excel("data_clientes.xlsx", sheet = "rep_generacion_cash")
datos$atraso_actual[is.na(datos$atraso_actual)]   <- 0   # ausencia de registro = 0
datos$n_empresas_sbs[is.na(datos$n_empresas_sbs)] <- 0
datos$deuda_sbs[is.na(datos$deuda_sbs)]           <- 0

vars_pred <- c("monto_aprobado","mora_promedio","atraso_maximo","atraso_max_3m",
               "atraso_actual","n_creditos_vigentes","deuda_compartamos",
               "n_empresas_sbs","deuda_sbs","ingresos","egresos","resultado_neto")

datos_z <- datos
datos_z[vars_pred] <- scale(datos[vars_pred])   # media 0, sd 1

# 2. Prueba M de Box (homogeneidad de covarianzas entre B, A, A+)
box_m_test <- function(data, group) {
  groups <- split(data, group)
  n_g <- sapply(groups, nrow); p <- ncol(data); n <- sum(n_g); g <- length(groups)
  S_list <- lapply(groups, function(d) cov(as.matrix(d)))
  S_pooled <- Reduce(`+`, Map(function(S, nj) S*(nj-1), S_list, n_g)) / (n-g)
  M  <- (n-g)*determinant(S_pooled, TRUE)$modulus -
        sum((n_g-1)*sapply(S_list, function(S) determinant(S, TRUE)$modulus))
  df <- (g-1)*p*(p+1)/2
  list(chi_sq = M, df = df, p_value = pchisq(M, df, lower.tail = FALSE))
}
box_result <- box_m_test(datos[, vars_pred], datos$especie)   # p < 0.00001

# 3. Modelo LDA de 3 grupos (2 funciones discriminantes: LD1, LD2)
modelo_lda <- lda(especie ~ ., data = datos_z[, c("especie", vars_pred)])

# 4. Evaluación: error de entrenamiento y LOOCV
pred <- predict(modelo_lda, datos_z[, vars_pred])
table(datos$especie, pred$class)                              # matriz de confusión
mean(datos$especie != pred$class) * 100                        # training error: 37.72%

modelo_lda_cv <- lda(especie ~ ., data = datos_z[, c("especie", vars_pred)], CV = TRUE)
mean(datos$especie != modelo_lda_cv$class) * 100                # LOOCV error: 62.28%

Línea a línea: las imputaciones de la línea 4–6 tratan la ausencia de registro como información real (sin atraso/deuda reportada) en lugar de un dato faltante al azar. scale() estandariza cada una de las 12 variables para que sus coeficientes en la función discriminante sean comparables en magnitud pese a tener escalas muy distintas (soles vs. conteos). La función box_m_test() implementa manualmente el estadístico M de Box: compara el logaritmo del determinante de la matriz de covarianzas combinada (S_pooled) contra la suma ponderada de los logaritmos de las matrices individuales por grupo; una diferencia grande (M alto) indica que las covarianzas no son homogéneas. Con CV = TRUE, lda() ejecuta automáticamente Leave-One-Out: cada cliente se clasifica usando un modelo entrenado con los 113 restantes, evitando el optimismo del error de entrenamiento.
